# Migration: v0.5.x → v0.6.0

v0.6.0 adds the **Pattern Layer** — distributable marketing/app blocks, spacing recipes, and `ai-patterns.json`.

## New exports

| Import | Purpose |
|--------|---------|
| `@ai-ds/core/blocks/HeroBlock` | Hero section |
| `@ai-ds/core/blocks/FeaturesBlock` | Feature grid |
| `@ai-ds/core/blocks/PricingBlock` | Pricing tiers |
| `@ai-ds/core/blocks/CTABlock` | CTA banner |
| `@ai-ds/core/blocks/FooterBlock` | Footer |
| `@ai-ds/core/blocks/LandingPageTemplate` | Full landing page |
| `@ai-ds/core/blocks/AppShellBlock` | Dashboard shell |
| `@ai-ds/core/blocks/AppSidebarBlock` | App sidebar |
| `@ai-ds/core/recipes` | `resolveRecipe`, `SPACING_RECIPES` |
| `@ai-ds/core/patterns` | Pattern manifest JSON |

## Install

```bash
npm install git+https://github.com/SCRUMUX/AICADS-PRO.git#v0.6.0
```

## Consumer Storybook

Consumer globs now include `node_modules/@ai-ds/core/blocks/**/*.stories.*`. Re-copy `.storybook/main.ts` from `templates/consumer-storybook/` or ensure your config uses `@ai-ds/core/storybook/createMainConfig` (updated in v0.6.0).

## Breaking changes

None for existing primitive imports. Pattern blocks are additive.

## Recommended workflow for new pages

1. Read `@ai-ds/core/patterns` → pick `pageTemplates.marketing.landing.default` or individual patterns.
2. Import blocks — do not rebuild sections with raw layout.
3. Use `LandingPageTemplate` for full landings.

See [docs/pattern-layer.md](../pattern-layer.md).
