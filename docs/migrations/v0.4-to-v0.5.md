# Migrating from `@ai-ds/core` 0.4 ŌåÆ 0.5

v0.5.0 is the **System Optimization Layer** release (Phase 10). The
internal surface gets stricter (token-driven, single Radix-aligned
scrollbar, CI guards), but only **two** consumer-visible breakages
remain:

| Change                              | Impact on consumers                                          |
| ----------------------------------- | ------------------------------------------------------------ |
| `<Toast>` custom primitive removed  | One-line API swap (sonner-backed `toast()` + `<Toaster>`)    |
| `<ScrollBar>` `@deprecated`         | Runtime warning today; full removal in v0.6.0                |

Everything else (tokens, contracts, ESLint shareable config, VRT, API
freeze, Figma plugin validation) is additive and does not require any
consumer code change.

## 1. `<Toast>` ŌåÆ `toast()` + `<Toaster>`

The custom presentational `<Toast>` JSX block has been **hard-deleted**.
The public Toast API is now the imperative sonner-backed surface.

### Before (0.4)

```tsx
import { Toast } from '@ai-ds/core/components/Toast';

<Toast appearance="info" title="Saved" body="Your changes are stored." />
```

### After (0.5)

```tsx
// Mount once near the app root:
import { Toaster } from '@ai-ds/core/components/Toast';
<Toaster position="top-right" />

// Anywhere in your code:
import { toast } from '@ai-ds/core/components/Toast';
toast.info('Saved', { description: 'Your changes are stored.' });
// Available variants: toast.info / .success / .warning / .danger
//                  + toast.message (neutral)
```

There is no behavioral difference for end users ŌĆö the sonner-backed
implementation already powered every interactive toast in v0.4.

## 2. `<ScrollBar>` ŌåÆ `<ScrollArea>`

`ScrollBar` is now marked `@deprecated`. Importing it triggers a
single `console.warn` at module load, and the AI manifest carries
`deprecated: true` so external tooling surfaces it. It will be
**removed in v0.6.0**.

### Before (0.4)

```tsx
import { ScrollBar } from '@ai-ds/core/components/ScrollBar';

<ScrollBar maxHeight={320}>{children}</ScrollBar>
```

### After (0.5)

```tsx
import { ScrollArea } from '@ai-ds/core/shared';

<ScrollArea style={{ maxHeight: 'var(--space-320)' }}>
  {children}
</ScrollArea>
```

`ScrollArea` is powered by `@radix-ui/react-scroll-area` and is already
used internally by `Dropdown`, `Autocomplete`, `Select`,
`CommandPalette`, `Modal` and `Drawer`.

## 3. Adopt the shareable ESLint config (optional, recommended)

v0.5.0 ships a consumer-side ESLint config that codifies the AICADS
isolation contract: consumer code must not import behavior engines
(Radix, cmdk, vaul, sonner, Floating UI, input-otp, cva) directly.

```js
// .eslintrc.cjs in your app
module.exports = {
  extends: [require.resolve('@ai-ds/core/eslint-config')],
};
```

After this, `import { Dialog } from '@radix-ui/react-dialog'` fails
with a clear message pointing at the AICADS semantic API.

## 4. No-op for the rest of your code

- All existing `@ai-ds/core/components/<Name>` imports keep working.
- All existing tokens, contracts and Tailwind preset paths are
  unchanged.
- Storybook stories under `playground/` still build with no changes
  to consumer apps.
- `ai-manifest.json` gained `deprecated`/`replacement` metadata and
  three new entries (`TooltipBubble`, `GridItem`, `Toaster`), but the
  schema is backwards compatible.

## 5. New developer workflows (informational)

These do not affect consumers ŌĆö they exist inside `@ai-ds/core`:

| Command                       | Purpose                                            |
| ----------------------------- | -------------------------------------------------- |
| `npm run lint`                | No-restricted-imports + `no-hardcoded-tokens` rule |
| `npm run manifest:check`      | Fails CI if `ai-manifest.json` drifts from source  |
| `npm run api:check`           | Fails CI if `etc/ai-ds-core.api.md` is out of date |
| `npm run figma:embed-manifest`| Refreshes the Figma plugin's embedded manifest     |
| `npm run chromatic` (playground) | Pushes Storybook baselines to Chromatic         |
